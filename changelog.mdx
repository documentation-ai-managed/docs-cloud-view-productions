<Update label="2024-08-10" description="v1.1.0" tags={["feature", "breaking"]}>

## New Features

- Launched API for programmatic tour creation: POST to `https://api.example.com/v1/tours`
- Added analytics webhooks for instant notifications on viewer milestones

```javascript
// Example webhook payload handler
app.post('/webhook', (req, res) => {
  const { event, data } = req.body;
  if (event === 'tour.viewed') {
    console.log(`Tour ${data.tourId} viewed by ${data.viewerIp}`);
  }
  res.status(200).send('OK');
});
```

## Breaking Changes

- Updated API authentication to require `Bearer {YOUR_TOKEN}` headers (legacy API keys deprecated)

</Update>

## How to Report Issues

<Callout kind="tip">

You help us improve by reporting bugs and suggesting features. Use the steps below for the fastest resolution.

</Callout>

<Steps>

<Step title="Check Existing Issues" icon="search">

Visit our [GitHub issues](https://github.com/example/cloudview-productions/issues) to see if your problem is already tracked.

</Step>

<Step title="Gather Details" icon="file-text">

Collect screenshots, browser console errors, and reproduction steps. Note your account email and property ID.

</Step>

<Step title="Submit Report" icon="send">

<Tabs>

<Tab title="GitHub" icon="github">

Create a new issue with clear title and steps.

</Tab>

<Tab title="Email" icon="mail">

Send details to `support@example.com` with subject: `[BUG] Brief Description`.

</Tab>

</Tabs>

</Step>

<Step title="Follow Up" icon="message-circle">

We'll respond within `48 hours`. Track progress via the issue thread.

</Step>

</Steps>
